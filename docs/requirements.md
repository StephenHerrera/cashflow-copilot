# Cashflow Copilot – Project Requirements

## Project Goal
The goal of this project is to build a budgeting application that helps users track their spending and understand their finances better.  
This project is also meant to help me practice backend APIs, basic AI concepts, and how a client and server work together.

## Target User
Students or early-career professionals who want a simple way to track expenses and see where their money is going.

## Planned Features (MVP)
- [ ] Add income and expenses manually
- [ ] Import transactions from a CSV file
- [ ] Automatically assign categories to transactions (AI-assisted)
- [ ] Set monthly budget limits for each category
- [ ] View a dashboard with totals and category breakdowns
- [ ] Get a basic forecast of end-of-month spending
- [ ] Detect unusual spending compared to past behavior

## Technical Goals
- Learn how to build and use REST APIs
- Understand how a frontend communicates with a backend
- Practice working with databases
- Apply basic machine learning techniques in a real project

## Non-Functional Requirements
- Use a local database during development (SQLite)
- Handle invalid inputs without crashing
- Keep the application reasonably fast and responsive

## Out of Scope (Version 1)
- Connecting to real bank accounts
- Advanced security features
- Mobile app support
- Multi-currency support
